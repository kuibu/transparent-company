## 中文

### 报告安全漏洞
请不要在公开 issue 中直接披露漏洞细节。

建议通过以下方式私下联系：
- GitHub Security Advisory（推荐）
- 或在 issue 中仅提交最小风险描述并标注“security”，等待维护者转私下沟通

### 报告内容建议
- 影响范围与前提条件
- 复现步骤（最小可复现）
- 可能影响（数据泄露/篡改/权限提升等）
- 修复建议（可选）

### 处理流程
- 维护者会先确认漏洞有效性与影响级别
- 评估修复方案与发布时间
- 修复后在发布说明中给出安全更新信息

## English

### Reporting a Vulnerability
Please do not disclose vulnerability details in a public issue.

Preferred channels:
- GitHub Security Advisory (recommended)
- Or open a minimal-risk issue labeled `security` and move details to private discussion

### Recommended Report Content
- Affected scope and preconditions
- Reproduction steps (minimal reproducible)
- Potential impact (data leak/tampering/privilege escalation, etc.)
- Suggested fix (optional)

### Response Process
- Maintainers validate impact and severity
- Define fix and disclosure plan
- Publish security notes in release/change log after remediation
